POST Multiple Series within Specific Years & Results

The code for this is similar to the last example, just with added parameters. To ask for additional information in your POST request, you’ll use a format similar to this: 

{
		{"seriesid":["ID1","ID2"],
		"startyear":"2000", 
		"endyear":"2010",
		"catalog":true,
		"calculations":true,
		"annualaverage":true,
		"registrationKey":"yourKeyHere"
		}
	}

Here are a few notes about making more specific POST queries:

  • The Series ID work the same as we’ve discussed.
  • The start and end year should be in a 4 digit format and can be a maximum of 20 year difference with Version 2 (10 years with Version 1).
  • The catalog , calculations, and annual average settings can be true or false depending on whether or not the you would like catalog data (this is only available on select queries which you can locate in the simple database where we searched for Series IDs if you would like additional calculations returned, or if you would like the annual average respectively.
  • Registration key is the same as in a simple POST request.

Results will be the same as in the earlier POST request as well, only now you should see results for the additional data you queried. Do keep in mind though, not all information will have this extra information available, and when working with the results, be prepared to sometimes not have all the info.
